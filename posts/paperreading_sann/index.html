Order Matters: Semantic-Aware Neural Networks for Binary Code Similarity Detection#

通读#

Abstract#

二进制代码相似度检测的目标是在不访问源代码的情况下检测相似的二进制函数，是计算机安全的一项基本任务。传统方法通常使用图匹配算法，这些算法速度慢且不准确。最近，基于神经网络的方法取得了巨大的成就。二元函数首先表示为具有手动选择的块特征的控制流图 (CFG)，然后采用图神经网络 (GNN) 计算图嵌入。虽然这些方法有效且高效，但它们无法捕获二进制代码的足够语义信息。本文中，我们提出了语义感知（Semantic-Aware）神经网络来提取二进制代码的语义信息。输入以基本块为单位的 CFG 图，首先使用 Bert 生成每一个基本块的 embedding，接着使用 MPNN 网络生成 CFG 图的整体 embedding，同时融入通过邻接矩阵生成的 CFG 图顺序信息，经过 MLP 得到最终的 embedding 值，以此进行相似度比较。本文在跨平台二进制代码分析任务、和二进制代码分类任务上都取得了很好的结果。

Introduction#

二进制代码相似度检测用于检测两个给定的二元函数是否相似。它有许多计算机安全应用，包括代码克隆检测、漏洞发现、恶意软件检测等。传统方法使用图匹配算法来计算相似度，但速度和性能堪忧。2017 年，Gemini 算法提出，输入一个二进制函数对，输出其相似度评分。其首先生成 CFG，使用人工设计的特征提取方式将其表示成低维特征向量，然后使用 Structure2Vec 计算得到其 embedding，然后通过 Siamese 网络计算相似度得分，效果优于传统方式。

但是，Gemini 仍存在一些问题。首先，CFG 图的基本块特征是一个 8 维向量，压缩时会导致大量语义损失；其次，二进制代码中节点顺序起着重要作用，Gemini 中未予以考虑。为解决此，本文提出一个具有三个组件的总体框架：语义感知模块、结构感知模块和顺序感知模块。

在语义感知模块中，我们使用 NLP 模型来提取二进制代码的语义信息。CFG 块中的标记被视为单词，块被视为句子。在先前的工作中，（Massarelli et al. 2019）使用 word2vec 模型训练块中的 token embeddings，然后使用注意机制获得 block embedding。（Zuo et al. 2018）借鉴了神经机器翻译（NMT）的思想来学习跨平台二进制码之间的语义关系。在本文中，我们采用 BERT（Devlin et al. 2018）对 tokens 和 blocks 进行预训练。与 BERT 相同，我们对 MLM（masked language model）任务的标记进行预训练，并提取所有相邻块对邻接节点预测任务（ANP）进行预训练。与分别学习 token 向量和 block 向量不同，本文方法能够同时学习 token 向量和 block 向量。此外，因为我们的最终目标是生成完整的图表示，所以我们添加了两个图级任务。一种是确定两个采样块是否在同一个图中，我们称之为图内块任务（BIG，block inside graph task）。另一种是区分块属于哪个平台/优化选项，称为图分类任务（GC，graph classification task）。我们发现，额外的任务能帮助提取更多的语义信息，并且能更好的学习块的表示，在对块嵌入进行预训练之后，我们将在图级任务上对它们进行微调。

在结构感知模块我们使用带有 GRU 模块的 MPNN，（Xu et al. 2018）已经证明了图神经网络可以具有像 Weisfeiler-Lehman 测试一样的区分能力。我们发现，在每个步骤中使用 GRU 比只使用 tanh 函数可以存储更多的信息。

在顺序感知模块，使用 CFG 的邻接矩阵作为输入，使用 CNN 来计算图顺序的 embedding。

本文有如下贡献：

  1. 提出了一个通用框架来学习 CFG 的 graph embedding
  2. 语义感知模块，使用 bert 预训练模型得到 block 的 embedding 值
  3. 顺序感知模块，使用 CFG 的邻接矩阵作为输入，使用 CNN 来计算图顺序的 embedding
  4. 对四个数据集的两个任务进行实验

Overall Structure#

以二进制代码函数的 CFG 作为输入，每个块是具有中间表示的标记序列。Semantic-aware 模块使用 bert 预训练模型得到 block 的 embedding 值，structural-aware 模块使用 MPNN 算法得到整个 CFG 图的 embedding 值，order-aware 模块借助 block 的邻接矩阵经过 CNN 提取得到了 CFG 图的结点顺序信息。最后，将 CFG 图 embedding 和结点顺序信息 concat 起来，经过 MLP 得到最终的表征。

Semantic-aware Modeling#

使用BERT预训练4种任务来处理CFGs。优点如下：

  1. 我们可以基于相同的模型从由不同平台、架构、编译选项的 CFGs 中提取 block embeddings。
  2. 我们可以通过预训练得到 token-level，block-level 和 graph-level 的信息。因为我们有 1 个 token-level 的任务，一个 block-level 的任务和两个 graph-level 的任务。
  3. 训练过程完全基于 CFG，不需要对编译器进行修改或其他的操作来获得相似的 block pair。

本文使用的 Bert 具有 4 个预训练任务，分别为：掩码语言模型 Masked language model（MLM）、相邻结点预测 Adjacency node prediction（ANP）、判断结点是否在图中 Block inside graph（BIG）和图分类任务 Graph classification（GC）。

MLM、ANP 是和原始 BERT 比较相似的两个任务。MLM 是 token-level 的，对 block 中的 token 进行 mask 操作，预测被 mask 的单词。ANP 是一个 block-level 的，因为 CFG 控制流图是一个有向图，有节点的拓扑顺序，因此可以将 CFG 中的所有相邻节点提取出来，作为相邻的“句子”，这些相邻的 block 对作为 ANP 任务的正例，并随机选择 CFG 图中不相邻的 block 对作为负例，预测后一个句子是否和前一个句子相邻（等价于 NSP 任务）。

BIG、GC 是 graph-level 级别的任务，目的是为了获取更多的图级别的信息。BIG 和 ANP 类似，区别是样本对的正例负例选择方式不同，BIG 任务的目的是让模型判断两个 block 是否在同一个图中，希望模型可以尽可能地学到此信息，从而对 graph-level 的层面有所帮助。因此，在 BIG 任务中同图的 block pair 为正例，不同图的 block pair 为负例。GC 是对 block 的分类任务，在本实验中不同平台、不同编译器、不同优化选项的条件下，得到的 block 信息有所不同，因此该任务希望模型可以让 block embedding 中包含这种信息，GC 任务的目标是判断某一特定的 block 属于哪个平台，哪个编译器，以及哪个优化选项。

Structural-aware Modeling#

得到 block 的 embedding 后，使用 MPNN 算法计算整个 CFG 图的 embedding 值。MPNN 全称是 Message Passing Neural Network（消息传递网络），是 google brain 2017 年的工作。MPNN 框架分为 3 个部分：消息传递、更新、和读出，在本文中消息传递使用 MLP，更新使用 GRU，读出使用求和函数来实现。

Order-aware Modeling#

该模块旨在提取 CFG 图中结点的顺序信息，首先构造结点的邻接矩阵，如下图所示，这里的邻接是有向的。有了邻接矩阵后，使用 CNN 进行特征提取。在图 a 的基础上，增加一个结点 1，但是三角形部分的结点顺序是不变的，对应矩阵中黄框的部分，可以视作一种平移不变性，图 c 相对于图 a 则是相当于一种伸缩不变性，结点顺序存在的这种特性可以被 CNN 捕捉到。

在我们的二进制代码相似性检测任务中，当在不同的平台上编译相同的函数时，节点顺序通常不会改变很多。大多数节点顺序更改是添加一个节点、删除节点或交换几个节点，因此 CNN 在我们的任务中很有用。除了学习节点顺序信息的准确性提高外，CNN 还具有一些额外的优势。首先，与传统的图特征提取算法相比，直接在邻接矩阵上使用 CNN 要快得多。其次，CNN 可以添加到不同大小的输入上，因此它可以在没有填充和裁剪等预处理的情况下对不同大小的图进行建模。

具体的 CNN 网络选择，使用 Resnet，因为 Resnet 的短连接可以更快、更高效的进行信息流动。使用 11 层的具有 3 个残差块的 Resnet，所有的特征映射都是 3 * 3 的，这样可以学习到图一些小的变化。$ g_o= Maxpooling (Resnet (A))$

Experiments#

Datasets#

本文在两个任务上进行实验。任务 1 为跨平台二进制代码分析，同一份源代码在不同的平台上进行编译，我们的目标是使模型对同一份源代码在不同平台上编译的两个控制流图 pair 的相似度得分**高于不同源代码 pair **的相似度得分。任务 2 为二进制代码分类，判断控制流图属于哪个优化选项。各数据集的情况如表 1 所示。任务 1 是排序问题，因此使用 MRR10 和 Rank1 作为评价指标。任务 2 是分类问题，因此使用准确率作为评价指标。
