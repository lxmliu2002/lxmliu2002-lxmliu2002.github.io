SSH 私钥泄露#

  • 信息探测

    1
2
3

    nmap -sV IP # 探测开放服务信息
nmap -A -v IP # 探测靶场全部信息
nmap -O IP # 探测靶场的操作系统类型与版本

  • 分析探测结果

    每个服务对应一个端口，用来进行通信。在扫描结果中查找特殊端口（大于 1024 的端口），针对特殊端口进行探测，尤其对开放大端口的 http 服务进行排查。

    • 对于 ssh 服务的 22 端口的靶场

      • 暴力破解
      • 私钥泄露

    • 对于开放 http 服务的 80 端口或其他端口的靶场

      • 对于开放的 http 服务的大端口，可以采用 http://ip:port/ 的形式访问

      • 查看源代码获取对应信息，如果没有 flag 信息，使用工具探测隐藏页面

        1

        dirb http://ip:port/ # 对这个服务的隐藏文件进行探测

        • 直接使用浏览器访问，查看效果、对应源代码，可能获取到 flag 或下一步对应的提示

        • 挖掘 ssh 信息，不断访问获取更多泄露的 ssh 密钥信息

          1
2

          url/.ssh/id_rsa # 得到私钥
url/.ssh/<id_rsa> # 得到认证关键字文件

          • 解密 ssh 密钥信息

            • 使用 ssh2john 将 id_rsa 密钥信息转换为 john 可识别的信息

              1
2

              chmod 600 id_rsa
ssh2john id_rsa > isacrack

            • 利用字典解密 isacrack 信息，得到密码

              1

              zcat /user/share/wordlists/rockyou.txt.gz | john --pipe --rules isacrack

          • ssh 连接主机

            1
2
3
4

            ls -alh # 查看 id_rsa 文件权限，需保证可读可写
chmod 600 id_rsa # 赋予文件可读可写权限
ssh -i id_rsa <username>@ip # username 通过查看认证关键字文件获取
# 输入上面解密得到的密码，即可远程登陆主机

          • 扩大战果

            1
2
3

            whoami # 查看当前用户

            • 如果不是 root 用户

              1
2
3
4

              cat /etc/passwd # 查看所有用户的列表

/tmp # 查看缓冲文件目录

            • 深入挖掘

              通过 /etc/crontab 文件，设定系统定期执行的任务，编辑需要 root 权限，不同的用户都可以有不同的定时任务。

              1

              cat /etc/crontab # 挖掘其他用户是否有定时任务，并查看对应的任务内容。执行的任务肯定对应靶场机器的某个文件

              如果在 /etc/crontab 下有某个用户的定时计划文件，但是具体目录下没有这个定时执行的文件， 可以自行创建反弹 shell，然后 netcat 执行监听获取对应用户的权限。如果有定时执行的文件，可以切换到对应的目录，查看对应的权限，查看当前用户是否具有读写权限。

文件上传漏洞 & 一句话木马#

文件上传流程：

  1. Client（浏览器）文件表单提交
  2. Server：生成临时文件
  3. 服务器端代码处理，如 PHP：读取临时文件信息，判断文件是否合规，保存文件

最常见的利用文件上传漏洞的方法就是上传网站木马（webshell）文件。WEBSHELL 又称网页木马文件，氛围 ASP 木马、PHP 木马、JSP 木马等，利用脚本语言中的系统命令执行、文件读写等函数的功能，一旦上传到服务器被脚本引擎解析，攻击者就可以实现对服务器的控制。

上传拦截，然后再修改文件名或修改 MIME 类型
