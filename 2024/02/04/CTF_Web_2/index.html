SQL 注入

SQL 注入是一种将 SQL 代码插入或添加到应用的输入参数中的攻击，之后再将这些参数传递给后台的 SQL 服务器加以解析并执行。攻击者把 SQL 命令语句作为输入被服务器 SQL 解释器正确解析执行，数据库把查询到的结果返回给服务器，然后呈现给攻击者，攻击者由此获得数据库内的数据信息。

产生条件：用户控制了 SQL 语句的一部分，用户的输入不再是一个输入参数，而成为了符合语法的 SQL 语句。

  • 有回显
    • 联合查询 -> 构造联合查询语句，直接查看查询结果
    • 报错注入 -> 构造报错语句，在报错中查看结果
    • 堆查询 -> 多行语句执行，今儿实现想要达到的目的
  • 无回显
    • 盲注 -> 布尔型 / 时间型 通过某种手段“爆破”结果

获取当前数据库的表名

MySQL 中把 information_schema 看作是一个信息数据库，保存着关于 MySQL 服务器所维护的所有数据库信息。

  • SCHEMATA 表：提供当前 MySQL 实例中的所有数据库信息，show databases 的结果来源。
  • TABLES 表：提供了关于数据库中的表的信息（包括视图），详细表述了某个表属于哪个 schema、表类型、表引擎、创建时间等信息，show tables from schemaname 的结果来源。
  • COLUMNS 表：提供了表中的列信息，详细表述了某张表的所有列以及每个列的信息，是 show columns from schemaname.tablename 的结果来源。

盲注

两大类型：

  • 布尔盲注 - 回显不同
    • 回显不同（内容、长度）
    • HTTP 响应状态码不同（200、500）
    • HTTP 响应头变化（无条件重定向、设置 cookie）
    • 基于错误的布尔注入（MySQL 是否报错）
  • 时间盲注 - 响应时间不同

布尔盲注

两大基本问题：

  1. 字符串的截取

    • substr() substring()

    • mid()

    • right()

      • 结合 ascii 使用

      • right 被过滤时考虑 left，结合 reverse 使用

    • trim()

      过滤指定字符串，最常见的用途是移除字首或字尾的空白。

      • trim()：删除前后空格
      • rtrim()：删除字符串结尾空格
      • ltrim()：删除字符串起始空格

      例如：

  2. 比较

    • 等于

    • 大于、小于

    • Like

      使用 % 表示任意字符

    • 正则表达式

      regexp rlike

      不区分大小写，需要大小写敏感的时候需要加上 binary 关键字

    • Between

      • between 筛选的是 expr >= begin_expr && expr <= end_expr 的数据，如果不存在则返回 0；

      • not between 筛选的是 expr < begin_expr && expr > end_expr 的数据，如果不存在则返回 0；

      • 如果 expr 返回 NULL，则 between 也返回的是 NULL

    • in

      in 也是大小写不敏感，字符和数字都可以用，也有 not in

    • 与、或、按位与、按位或

    • 异或、按位异或

      xor 为异或，^ 为按位异或
