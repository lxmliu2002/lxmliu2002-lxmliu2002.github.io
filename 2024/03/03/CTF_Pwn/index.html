CTF_Pwn

XMCVE 2020 CTF Pwn入门课程

概述

  • 破解、利用成功（程序的二进制漏洞）
  • 攻破（设备、服务器）
  • 控制（设备、服务器）

名词：

  • exploit：用于攻击的脚本与方案
  • payload：攻击载荷，是对目标进程被劫持控制流的数据
  • shellcode：调用攻击目标的 shell 的代码

一、二进制基础

程序的编译与链接

编译：由 C 语言代码生成汇编代码

汇编：由汇编代码生成机器码

链接：将多个机器码的目标文件链接成一个可执行文件

Linux 下的可执行文件格式 ELF

分类：

  • Windows：
    • 可执行程序：PE
    • 动态链接库：.dll
    • 静态链接库：.lib
  • Linux：
    • 可执行程序：ELF
    • 动态链接库：.so
    • 静态链接库：.a

ELF 文件结构：

  • ELF 文件头表（ELF header）：记录 ELF 文件的组织结构
  • 程序头表/段表（Program header table）：
    • 告诉系统如何创建进程
    • 生成进程的可执行文件必须拥有此结构
    • 重定位文件不一定需要
  • 节头表（Section header table）：
    • 记录 ELF 文件的节区信息
    • 用于链接的目标文件必须拥有此结构
    • 其他类型目标文件不一定需要

磁盘中的 ELF（可执行文件）与内存中的 ELF（进程内存映像）

进程虚拟地址空间

地址以字节编码，常以 16 进制表示

虚拟内存用户空间每个进程一份，内核空间所有进程共享一份，mmap 段中的动态链接库仅在物理内存中装载一份

段与节：

  • 一个段包含多个节
  • 段视图用于进程的内存区域的 rwx 权限划分
  • 节视图用于 ELF 文件变异链接时与磁盘上存储时的文件结构的组织
  • 代码段包含了代码与只读数据
  • 数据段包含了可读可写数据
  • 栈段

大端序：低位放在高地址

小端序：低位放在低地址

程序的装载与进程的执行

X86 & amd64 汇编简述

二、栈溢出基础

C 语言函数调用栈

用来保存函数运行时的状态信息，包括函数参数与局部变量等。发生函数调用时，调用函数的状态被保存在栈内，被调用函数的状态被压入调用栈的栈顶。函数调用结束时，栈顶的函数状态被弹出，栈顶恢复到调用函数的状态。在内存中从高地址向低地址生长，栈顶的地址压栈时变小，退栈时变大。

主要涉及三个寄存器——esp（栈顶）、ebp（栈底）、eip（程序指令地址）。

  • 调用：

    • 首先将被调函数的参数逆序压栈，然后将函数的返回地址压栈。

    • 将被调用函数的局部变量压栈

  • 恢复：

    • 弹出局部变量，栈顶指向被调用函数的基地址
    • 然后将基地址内存储的调用函数的基地址从栈内弹出，存储到 ebp 内，从而恢复调用函数的 ebp，栈顶指向返回地址
    • 将返回地址弹出存到 eip 中（尝试覆写返回地址）

ret2text

缓冲区溢出：向定长的缓冲区写入超长的数据，造成超出的数据覆写了合法内存区域

  • 栈溢出：最常见、漏洞比例最高、危害最大的二进制漏洞
  • 堆溢出
  • BSS 溢出：攻击效果依赖于 BSS 上存放了何种控制数据
